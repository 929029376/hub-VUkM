## 1. 什么是前后端分离？

**前后端分离**是一种软件架构模式，将前端（用户界面）和后端（业务逻辑、数据处理）完全独立开发和部署。在这个项目中体现为：

### 项目中的三层架构：
1. **模块1 - MCP Server** (`main_mcp.py`)
   - 提供工具服务层
   - 运行在端口 8900
2. **模块2 - FastAPI 后端** (`main_server.py`)
   - 提供 HTTP API 接口
   - 运行在端口 8000
   - 包含路由层（routers）、服务层（services）、数据模型层（models）
3. **模块3 - Streamlit 前端** (`demo/streamlit_demo.py`)
   - 用户交互界面
   - 通过 HTTP 请求调用后端 API

### 历史对话存储机制：
项目使用**双层存储策略**：
#### **第一层：关系型数据库（SQLite）**
#### **第二层：OpenAI-Agent Session 存储**

###  历史对话作为大模型输入的完整流程：
#### **步骤1：初始化会话**
#### **步骤2：用户发送新消息时**
#### **步骤3：调用大模型**
#### **步骤4：保存助手回复**
#### **步骤5：前端获取历史对话**
